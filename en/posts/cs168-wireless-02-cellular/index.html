蜂窝网络

为什么学习蜂窝网络？

无线移动连接已经成为现代标准。即使坐在行驶的汽车里，手机也能连接互联网。

传统互联网无法支持这种能力。你也许能从卧室走到厨房而不断网，因为你仍在家用无线路由器的覆盖范围内，而路由器再通过线缆连接互联网。但传统互联网不能在很长距离的移动中，例如乘车时，提供无缝连接。

无线移动连接有许多实现方式，但蜂窝网络是当今占主导地位的接入技术。如今超过一半的 Web 流量来自蜂窝设备。

蜂窝网络只是移动无线连接技术之一。卫星、自由空间光通信等技术同样存在，不过蜂窝网络仍是主流。

未来，自动驾驶、虚拟现实等依赖无线移动技术的高性能应用可能推动更多创新。若要扩展现有蜂窝网络来支持这些应用，成本可能高得难以承受；而 AT&T、Verizon 等蜂窝运营商也不以快速创新闻名。普遍看法是，这个领域很可能在不久的将来受到颠覆，因此也是活跃的研究方向。

蜂窝网络简史

蜂窝技术源自传统电话系统。它最初让用户摆脱有线座机，无线拨打电话。第一部移动电话于 1983 年售出，价格为 4000 美元；考虑通胀后，今天会贵得多。

蜂窝技术来自电话网而非互联网，因此许多设计选择与传统互联网不同。多年来，蜂窝技术（例如智能手机出现前用于语音通话的手机）与互联网并行发展，两者采用了不同的架构选择。

例如蜂窝网络采用资源预留，现代互联网采用分组交换；蜂窝网络常以单个用户为单位思考，互联网主要以流或分组为单位；蜂窝网络可按通话分钟计费，而互联网通常不会如此细致地记录用量。

近年蜂窝网络逐渐与传统互联网兼容。今天可以把它看成一种专用的第 2 层局域网，能够与传统 TCP/IP 互联网的其他部分交互。

蜂窝标准

传统互联网由标准组织帮助统一 TCP、IP 等协议，蜂窝网络也由多个标准组织合作制定标准。

在某些方面，蜂窝标准组织面对的现实政治复杂性更高。为了互操作，所有手机制造商和网络运营商（例如建设基站的 Verizon）必须从物理层开始就对协议达成一致。

蜂窝领域的关键标准组织是 3GPP（第三代合作伙伴计划），大型设备供应商与电信公司都参与其中。3GPP 提出标准，再提交给 ITU（国际电信联盟）。ITU 隶属联合国，每个国家都有一票，因此标准制定也涉及政治。例如每国一票意味着美国可能被欧盟国家的票数压过。

通常每十年引入一代新技术，这就是 2G、3G、4G、5G 中数字的含义。5G 在 2020 年前后定义，运营商仍在部署；6G 标准的规划会在 2020 年代后期开始。

每一代都从多个维度改进上一代，包括峰值理论速率、用户平均速率、移动性（高速移动时保持连接）、连接密度（指定区域内设备数量）等。各维度通常都比上一代提高约十倍。

除性能外，架构也逐代从电话网设计转向互联网设计。1G 手机完全是模拟的，面向语音通话；2G/3G 仍以电路交换和语音为主，辅以少量短信，互联网流量很少；4G 起转向分组交换，语音只是网络承载的众多应用之一。

蜂窝规范有数千页、数百份文档，几乎没人会完整读完。更麻烦的是，每次换代几乎所有组件都会改名。例如蜂窝塔曾依次叫 base station、NodeB、evolved NodeB（eNodeB）、next-gen Node B（gNB），其实都指同类组件。本课会自行采用更直观的名称。教材或规范可能使用不同术语，但概念总体一致。

关键挑战：移动性

蜂窝网络最难处理的是移动性。可以把它想成手机在高速公路移动时仍播放视频（当然不要边开车边看）。我们会研究四个根本挑战：

  1. 发现：移动过程中，如何知道应连接哪个蜂窝塔？
  2. 认证：AT&T 的塔可能只愿为自家客户提供连接，如何确认用户身份？
  3. 无缝通信：离开一座塔、进入另一座塔覆盖范围时，连接应不中断。
  4. 问责与计量：客户只购买 6 GB 流量时，超额后网络应停止连接或降低服务质量。这继承自按通话分钟计费的旧蜂窝网络，也因为蜂窝资源稀缺而保留。

基础设施组件：无线电塔

蜂窝网络的第一个组件是无线电塔。塔上有天线；内部的无线电收发器把数字比特转换为经空中接口传送的模拟信号；无线电控制器则决定如何分配无线资源。

可以把控制器想成运行调度器的 CPU。它根据需求和商业模式，例如客户付费多少，把不同频段和时段分给不同客户。这其实是很困难的调度问题，这里不再展开。

下面是控制器分配资源的简化模型。每个彩色矩形表示某用户（用颜色区分）可在特定时刻使用特定频率。

每条竖截面代表一个时隙，显示该时隙内频率如何分配。例如第一个时隙中，蓝色用户得到 3 个频率槽，橙色用户 5 个，灰色用户 4 个。

每条横截面代表一个频率，显示它随时间如何分给用户。例如最上行的频率先给灰色，后来依次给绿色、蓝色、红色等。

这个模型通过资源预留而非尽力而为来共享资源。用户只能在控制器分配给自己的频率和时间发送。

传统上无线电控制器安装在塔内或附近；如今也有人把控制器迁移到云端，以便维护和管理。

每个运营商在全国布置许多蜂窝塔，让用户无论身在何处都能连接，最终形成无线接入网（RAN）。

通常每座塔有自己的一组可用频率，相邻塔使用不同频段，从而避免互相干扰。图中每种颜色代表一组频率；两座塔可以都使用蓝色频段，只要它们不相邻。任何相邻塔的频率都不重叠。频率也常按需求分配，例如旧金山市中心的塔会比荒郊野外的塔获得更多频率。

基础设施组件：蜂窝核心网

移动用户把数据发送到蜂窝塔后，塔还要把它送往互联网。每座塔通过有线链路接入蜂窝核心网，可把核心网理解为不直接面向用户的后端基础设施。

核心网包含数据平面组件，类似普通路由器与交换机，在用户（经塔）和网络其他部分之间转发分组。这里关注两类特殊路由器。

无线网关是 RAN（蜂窝塔）与核心网的边界，塔把数据转给其中一个无线网关。核心网另一端的分组网关是蜂窝网络与互联网的边界；用户数据最终抵达分组网关，并作为标准 TCP/IP 分组发往互联网。

核心网还有传统互联网中没有的控制平面组件，用户流量不会经过它们。这里关注其中两个。

数据库保存客户信息：客户有哪些设备、订购何种套餐、设备目前在哪里（例如连着哪座塔）。

移动性管理器是管理网络功能的控制器，可把它想成 CPU。它帮助认证用户，例如确认对方确实是 Verizon 客户，也会随着用户移动更新配置。

总的来说，用户设备向 RAN 中的塔发送数据；塔把数据转给无线网关进入核心网；数据最终到达分组网关，再转入互联网。核心网中还有移动性管理器、数据库等控制组件，用于保存和管理客户信息。

蜂窝网络运行的高层步骤

步骤 0，注册。用户登记蜂窝服务，例如去 Verizon 门店购买套餐、签订合同；运营商把用户与套餐信息存入数据库。

步骤 1，发现。用户在某处打开手机，手机必须发现附近有哪些塔，并选择一座。

步骤 2，附着。设备选好塔后请求连接。塔必须询问移动性管理器是否允许，例如用户是否已超出配额。

若认证通过，管理器配置塔和路由器，建立一条经塔与路由器从用户通往互联网的路径。

步骤 3，数据交换。用户现在可沿配置好的路径收发数据。

步骤 4，切换。用户移动时可能远离原塔、接近同一运营商 RAN 中的新塔。旧塔、新塔和设备共同决定是否切换。

各方同意后通知移动性管理器，由它重新配置塔和路由器，经新塔（也可能经不同路由器）建立新路径。切换必须无缝：整个过程中用户仍可能收发数据，不应受到干扰。为此，网络必须持续紧盯用户设备。

用户不断移动、最佳塔不断变化时，步骤 3 与 4 会反复进行。

最后还要实现漫游。用户去德国时，美国运营商 Verizon 可能没有覆盖，但它可以与德国电信签约，让自家客户使用后者的基础设施。德国电信因而既要服务自家用户，也要服务 Verizon 等其他网络的用户。

在访客网络中连接的步骤总体相似，只是访问地网络和归属网络的移动性管理器还须协调，例如德国电信向 Verizon 确认用户是否购买漫游服务。

步骤 0：注册

登记套餐时会获得 IMSI（International Mobile Subscriber Identity，国际移动用户识别码），这是与订阅绑定的唯一标识，安全地存放在 SIM 卡硬件中。

这就是运营商给你 SIM 卡的原因。换手机但不换套餐时，只需把 SIM 卡移到新手机，新手机便关联同一 IMSI；换套餐但继续用原手机时，换一张 SIM 卡，手机就关联新的 IMSI。

IMSI 前 3 位是标识国家的移动国家码；接下来 2 至 3 位是代表服务商（如 Verizon、AT&T）的移动网络码；剩余位是移动用户识别号，在该服务商内部标识具体用户。IMSI 总长不超过 15 位。

IMSI 不等于 IP 地址。购买一年套餐后，全年 IMSI 不变；但每次附着并连接网络时，都可能得到不同 IP 地址。

蜂窝网络还使用两个不同于 IMSI 的标识，这里不深入讨论。IMEI（International Mobile Equipment Identity，国际移动设备识别码）唯一标识物理设备，编码制造商和型号，例如“这是一部 iPhone 13”，换套餐后仍不变。同一套餐覆盖两部手机时，会有两个 IMEI，但只有一个 IMSI。

另一个标识是电话号码，它同样不同于 IMSI、IMEI，各位数字含义也不同，例如区号。电话网需把电话号码与特定 IMSI 关联，才能确定对应套餐。

注册并取得 IMSI 后，运营商把 IMSI 和套餐信息存入数据库。

注册时，用户设备（SIM 卡）和运营商（数据库）还约定一个共享密钥，附着时会用到。

步骤 1：发现

设备如何发现覆盖范围内、属于自己运营商的塔？

每座塔周期性发送信标（问候消息），告诉覆盖范围内的所有设备自己存在。信标也包含运营商，例如“我是 Verizon 的塔”，运营商用 2 至 3 位移动网络码标识。SIM 卡上的 IMSI 同样含移动网络码，设备因此可比较：“我的 SIM 属于网络 220，这座塔也声明网络 220，所以可以使用。”

信标在专用的控制信道频率上传输，以免干扰数据。每个频段都有对应控制信道；相邻塔使用互不重叠的频段，也就使用不同控制信道，从而避免干扰。

设备可能听见多个信标。它测量不同塔的信号强度，从属于本运营商的塔中选择信号最佳者。

还有一个自举问题：设备如何知道应监听哪个控制信道？它必须先调到控制信道，才能接收信标。

解决办法有几种：设备可以扫描并尝试很多频率，虽慢却有时是唯一选择；运营商可在注册时预置控制信道列表；设备也可缓存以前用过的信道。

发现后的后续塔扫描并非必要。切换时，旧塔会准确告知设备新塔所用的数据频率。因此切换约需 0.01 至 0.1 秒，远快于发现阶段扫描所需的 10 至 100 秒。

步骤 2：附着

  1. 用户发现一座塔后，向它发送附着请求，并在请求中带上 IMSI（用户标识）。
  2. 塔把请求发送给真正负责处理的移动性管理器。
  3. 管理器用 IMSI 查询数据库，获取套餐详情；还利用设备和数据库共同掌握的密钥执行认证（省略密码学细节）。

认证成功说明用户身份真实；数据库又显示用户有权使用服务时，管理器批准附着请求。

  1. 请求获批后，管理器配置数据平面。它先给设备分配 IP 地址，再配置塔，告诉无线电控制器为该用户分配多少资源；随后配置塔与路由器，建立设备到互联网的路径；最后初始化计数器和流量整形器，记录设备的互联网用量。

连接建立后，管理器把用户位置写入数据库，具体来说是把 IMSI 映射到 IP 地址及当前路径（所用塔和网关）。

整个附着过程都在控制信道上进行。用户尚未获分配数据频率，只能用专用控制信道通信。

步骤 3：数据交换

此时网络已配置完毕，设备可用自己的 IP 地址收发消息。

蜂窝网络中的塔、无线网关和分组网关如何转发？用户不断移动，若运行距离向量等传统路由算法，路由将永远无法收敛。

因此，管理器用隧道建立设备到互联网的路径。分组依次经过设备、塔、无线网关和分组网关。

概念上，我们告诉塔：“收到该用户的分组，就沿此方向送入蓝色隧道。”分组在有线链路另一端离开蓝色隧道，到达无线网关；再告诉无线网关：“收到从该隧道出来的分组，就沿此方向送入绿色隧道。”分组穿过绿色隧道到达分组网关，由后者转入互联网。

入站分组也走隧道。告诉分组网关：“收到发往用户 A 的分组，就放入通向无线网关的绿色隧道”；再告诉无线网关：“收到离开绿色隧道的分组，就放入通向塔的蓝色隧道。”

这些组件都不运行路由协议来寻路，而是由管理器直接告知转发方式。每个用户需要自己的一组隧道，因此网络保存每用户状态，例如每个在线用户一条表项。

规则如何实现？例如无线网关怎样知道入站分组来自蓝色隧道？可以使用封装：进入隧道时增加一个首部，注明分组正通过哪条隧道；分组在另一端离开时，网关读取额外首部，得知来源隧道，并据此决定下一跳。

使用隧道和封装后，路由器从不依据用户 IP 转发。用户一直移动，IP 无法表示位置；必须由预配置隧道决定转发方向。

步骤 4：切换

用户从一座塔移到另一座时会怎样？下面看一个稍作简化的协议，把两座塔称为旧塔与新塔。

  1. 设备持续测量与旧塔间的信号强度并上报。旧塔发现信号过弱时，给出同一运营商附近塔及其控制信道频率，让设备测量它们。
  2. 设备测量邻近塔信号并上报旧塔；旧塔按运营商策略选择最佳新塔。
  3. 旧塔通知新塔用户即将到来，使新塔为该用户分配频率资源。
  4. 新塔把已分配的频率资源告知旧塔。
  5. 旧塔通知用户按这些频率连接新塔。
  6. 新塔向移动性管理器报告自己已成为该用户的新塔。管理器更新用户位置，并更新隧道，经新塔以及可能不同的无线网关、分组网关建立新路径。
  7. 最后，新塔通知旧塔切换完成。

切换如此复杂，是因为要让用户在塔间移动时无缝通信、完全不中断。这需要用户、两座塔、移动性管理器和网关共同协作。

无缝通信困难还因为切换不是原子操作，过程中用户仍在收发数据。例如外部服务器的回复可能已经把一批入站分组发往旧塔；切换期间旧塔继续缓冲该用户的数据，完成后再把缓冲数据送给新塔，由新塔转给用户。传统 TCP/IP 网络不需如此缓冲；这是为移动中的无缝切换新增的能力。

切换决策始终由运营商作出，设备不能自行选择下一座塔。好处是运营商控制力更强，例如某塔过载时可把用户负载均衡到另一座塔，也可把低优先级用户送往较差的塔；代价是更慢，需要更多往返和更高复杂度。

切换期间用户 IP 地址不变，只更新隧道，让发往该 IP 的分组改走另一条路径。

切换复杂且需要更新网络中的每用户状态。用户数量增加或移动极快时，协议会遇到扩展性挑战。现代蜂窝网络仍能在大规模下良好运作，是因为这些协议经过了大量优化；这也解释了标准为何动辄数千页。

漫游

用户在国外或自家运营商没有覆盖的地方，可以漫游并连接其他网络。

在访客网络中的发现、附着与切换和归属网络总体类似，主要差别是访客网络的移动性管理器必须与归属网络的管理器通信。

例如访客网络需要归属网络协助认证用户、确认是否购买漫游；还要把跟踪数据发回归属网络，让它知道用户位置。

访客网络如何知道归属网络在哪里？附着时设备会提交 IMSI，其中的移动网络码标识用户运营商。

用户到互联网之间可用两种隧道方案。

归属路由（home routing）让流量穿过归属网络的分组网关。所有分组必须从访客网络返回归属网络，再转往互联网。好处是归属网关可跟踪用户；缺点是美国用户在德国漫游并访问德国网站时，分组得从德国去美国网关，再返回德国。

本地疏导（local breakout）让流量穿过访客网络的分组网关，分组无需先回归属网络，路径更短；但用量核算更复杂，因为漫游网络必须计量并把数据发回归属网络。

其他操作

前面介绍了蜂窝网络的关键操作，实际还存在其他操作。

合法监听（lawful intercept）是所有蜂窝运营商必须满足的法律要求，使持有搜查令的政府能够监听连接和用户发送的分组。

失窃手机登记库允许用户报告手机被盗。窃贼尝试让失窃手机接入网络时，运营商的管理器和数据库会发现它被盗，并尝试定位。运营商依据硬编码在手机中的设备标识 IMEI 来识别具体手机，不受用户标识 IMSI 影响。设备接入时必须报告 IMEI，运营商即可检查是否失窃。

这些操作之所以可行，是因为运营商采用集中控制，掌握所有用户及其位置。

对蜂窝网络设计的反思

蜂窝网络与传统互联网有不同的根本目标和设计选择。认证与计量是蜂窝网络的核心目标，却不是传统互联网的目标；资源按预留分配，网络还维护动态变化的每用户状态。

这种有状态、基于预留的网络增加了复杂性。用户移动时，各组件必须持续重新配置隧道。

可以考虑其他设计。切换复杂，是因为希望用户移动时保留同一 IP。如果每次切换都改变 IP，地址就能真实反映用户位置，也可重新采用传统路由协议；但 TCP、HTTP 等高层协议会中断，因为 TCP 依赖通信双方保持相同 IP 地址。

保留 IP 增加复杂度，改变 IP 又会破坏 TCP。一种可能的方案是使用允许 IP 地址变化的传输层协议，例如 Google 开发的 QUIC。这样即使 IP 改变，也能用 IPv6 首部的流标签字段标记同一条流中的所有分组。
