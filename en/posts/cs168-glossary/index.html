CS168 术语表

CS168 计算机网络课程术语表，覆盖分层、转发、路由、寻址与可靠传输。

July 12, 2026
Edit post

本文译自 Glossary — CS 168 Textbook ，依照 CC BY-SA 4.0 许可发布 。 中文完整翻译；保留英文术语以便对照。原文作者为 Peyrin Kao，课程内容基于 Sylvia Ratnasamy、Rob Shakir 等人的讲授；原术语表改编自往期 CS 168 课程。

术语表

本术语表改编自 往期 CS 168 课程。

术语定义
ACK（确认）

设置了 ACK 标志、表示数据已经收到的 TCP 分组。

ARP（地址解析协议）

让设备在 IP 地址与 MAC 地址之间建立映射的协议。设备广播 ARP Request，查询某 IP 对应的 MAC；被查询设备以单播 ARP Response 回复。映射缓存在 ARP 表中，表项会超时，属于软状态。

Autonomous System（自治系统）

由单一实体或组织统一管理的一张或一组网络。一个 ISP 往往是一个 AS，也可能把网络划为多个 AS。每个 AS 有一个编号，BGP 用它标识路径。

AXE

一种拟议的 STP 替代方案，用重复抑制而非生成树来避免环路，也是世上一些绝妙诗句的来源。

分组可能遭遇的坏事

丢失、损坏、乱序、延迟、重复。

Bandwidth-Delay Product（带宽时延积）

带宽与传播时延的乘积，表示“填满管道”所需的比特数，即发送方以链路带宽发送时，已经发出但尚未收到的比特数。

Bellman-Ford Equation（贝尔曼—福特方程）

到目的地的最短距离，是对所有邻居取“到邻居的代价加邻居到目的地距离”的最小值。具体为：d(u,v) = min（邻居 w）[c(u,w) + d(w,v)]。

Best Effort（尽力而为）

按需交付；除系统会尽最大努力外，不提供任何性能保证。

Border Router（边界路由器）

与其他网络中的路由器相连的路由器。

Checksum（校验和）

用于检测损坏的数值，由分组的某一部分计算得到；具体覆盖范围取决于协议。

CIDR（无类别域间路由）

把 IP 地址与网络掩码结合，用于确定哪些位属于网络位，比最初的有类别 IP 寻址灵活得多。

Circuit Switching（电路交换）

端系统沿路径预留带宽、建立电路后通信的数据传输方式，不需要分组。

Classful Addressing（有类别寻址）

确定 IP 地址网络位与主机位的方案。A 类以 0 开头，前 8 位标识网络、后 24 位标识主机；B 类以 10 开头，前后各 16 位；C 类以 110 开头，前 24 位标识网络、后 8 位标识主机。

Control Plane（控制平面）

用于计算路由表和其他转发信息的网络机制。

Convergence（收敛）

当各方都掌握最新信息，且网络拓扑不变时，之后收发的更新都不再改变路由状态，就称算法已经收敛。

Core/Backbone Router（核心/骨干路由器）

与其他内部路由器相连的路由器。

Cost Table（代价表）

路由器上保存到所有邻居代价集合的数据结构。

Count-to-Infinity Problem（计数到无穷问题）

距离向量信息异步传播可能造成的一类路由环路。链路失效后，原来使用故障路径的路由器误以为邻居仍有有效路径，在不知道该路径包含原故障路径一部分的情况下采用它；两个邻居反复接收对方更新并采用这条坏路径。

Cumulative ACK（累积确认）

表示“直到这个分组（或字节）为止，我都已收到”的 ACK。

Data Plane（数据平面）

用于转发数据的网络机制。

Datacenters（数据中心）

规模巨大的机器集合。

David Clark（戴维·克拉克）

互联网未被充分歌颂的英雄，曾任首席架构师，并提出端到端原则。

Dead End（死路）

分组到达路由器或交换机后，转发决策找不到输出端口，只能丢弃。

Destination-Based Routing（基于目的地的路由）

只依赖目的地址的路由。两个不同源到同一目的地的路径一旦重合，之后必须一致。

DHCP（动态主机配置协议）

主机接入网络时为其提供 IP 地址的协议。主机广播 Discovery；服务器用 Offer 提供 IP、子网掩码、第一跳路由器 IP 和租期；主机发送 Request 选择提议；服务器以 Acknowledgement/Acceptance 确认。所有 DHCP 消息都广播。

Distance Vector Routing（距离向量路由）

可扩展的分布式算法。每台路由器保存到各目的地的距离“向量”和下一跳，把最短距离向量泛洪给邻居；收到向量后，用 Bellman-Ford 更新自身向量。

DNS（域名系统）

把名称与地址关联的系统，常用于由主机名查询主机地址。

Dotted-quad notation（点分四组表示法）

把 IPv4 地址写成四个数字、每字节一个数字的表示法，例如 12.34.158.5。

Duplicate ACKs（重复 ACK）

多次确认同一已收数据的一串累积 ACK，是孤立丢包的信号，因为额外 ACK 表明后续数据仍在抵达。

Edge Router（边缘路由器）

端主机直接连接到的路由器。

End-to-End Principle（端到端原则）

帮助判断功能应放在网络内还是只放在端主机。三种解释：仅在必要时，主机能实现就不在网络中实现；仅在充分时，只有该层能完整实现且可减轻主机负担时才实现；仅在有用时，若能改善性能又不给不需要它的应用增加负担，就在网络中实现。

Enterprises（企业网络）

公司和大学。

Fate Sharing（命运共享）

把状态存放在依赖该状态的实体中，使该实体不受其他故障影响。

First-hop Router（第一跳路由器）

主机向第 2 层网络之外的目的地发送分组时，首先把分组交给的路由器。

Flooding（泛洪）

本课指交换机把分组从除入端口外的所有端口发出。

Flow（流）

两个进程之间的一串分组。

Forwarding（转发）

把分组送向目的地：读取首部地址，在路由状态中查找正确输出端口，再由该端口发出。它是路由器内数据平面的局部过程，必须快速完成。

Forwarding Entry（转发表项）

把一个地址或一组地址映射到输出端口的转发表条目。

Forwarding Table（转发表）

路由器为指导转发决策而计算的表，利用邻居表和代价表中的信息生成。

Fragmentation（分片）

把分组拆成更小的分组，以适应链路的最大传输单元（MTU）。

Full-information ACK（全信息确认）

描述迄今收到的全部数据，例如“直到此分组都已收到，此外还收到这些分组”。

Hard State（硬状态）

信息不会超时的系统：一旦获得某项知识，就假定它一直真实有效，直到收到明确的相反通知。

Host bits（主机位）

IP 地址中标识该网络内部主机的部分。

Host/End System（主机/端系统）

网络端点，负责产生随后由网络路由的数据分组。

Individual ACK（单独确认）

表示“我收到了这一个特定分组”的 ACK。

Internet（互联网）

连接所有联网计算设备的核心网络基础设施。

IP Address（IP 地址）

第 3 层使用的寻址方案。

IPv4

IP 协议的第 4 版。

ISP/ISP Network（互联网服务提供商/ISP 网络）

由分组交换机和通信链路组成、为端系统提供网络接入的网络。

LAN（局域网）

覆盖较小地理区域的第 2 层网络，例如一栋住宅。

Layering（分层）

把复杂系统拆成彼此构建或依赖的层次。互联网中特指物理层 L1、数据链路层 L2、网络互联层 L3、传输层 L4；每层只与直接上下相邻层交互。

Layers（各层）

应用层：为应用提供网络支持；4 传输层：可靠或不可靠的端到端交付；3 网络层：全局尽力交付；2 数据链路层：本地尽力交付；1 物理层：在介质上传输比特。

Learning Switches（学习交换机）

通常在第 2 层与生成树协议配合使用。它维护目的地到输出链路的转发表，从分组源字段学习。收到分组后，若目的地在表中就由对应链路转发，否则泛洪。

Linecard（线卡）

路由器中收发分组的硬件，更新校验和、TTL 等字段并选择输出端口。

Link（链路）

连接路由器的物理基础设施。

Link-State Routing（链路状态路由）

每台路由器用协议专用的广播机制把自身链路状态发给网络中所有路由器，使每台路由器学到完整网络图，再用任意有效算法（如 Dijkstra）计算自己到所有节点的最低代价路径。

Loop（环路）

分组永远在同一组节点间循环。

LPM（最长前缀匹配）

IP 地址匹配多个前缀时选择最长者；可想成遍历前缀树，直到地址无法继续匹配。

MAC Address（MAC 地址）

用于第 2 层路由的 48 位数，写入主机和路由器的网络接口硬件，只读存储使其成为永久标识。

Maximum Transmission Unit, MTU（最大传输单元）

链路能作为一个单元传输的最大比特数，即可跨链路发送的最大分组大小。

Modularity（模块化）

把问题分解为任务或抽象，由此产生分层设计原则。

Multihoming（多宿主）

把一台主机接到多个彼此独立的网络，使一个上游网络离线后主机仍可达；它会阻碍聚合。

NACK（否定确认）

“我没有收到预期数据”的消息。

Network（网络）

非正式用法指由端系统、路由器/交换机和链路组成，能在主机间传送数据的系统，例如伯克利校园网。正式用法指 IPv4 中共享同一网络地址的一组网络元素，常与子网同义。

Network Address（网络地址）

IP 地址中指向网络或子网、而非主机的部分。

Network bits（网络位）

IP 地址中标识主机所在网络的部分。

Network mask（网络掩码）

形似 IP 地址的比特串，用于标识 IP 地址中的网络部分；由固定数量的 1（每个网络位对应一个）后接全 0 构成。

Network Name（网络名称）

主机的名称，通常便于人类理解。

Network Stack（网络栈）

主机上的网络软件，复现路由器中的部分功能，并增加套接字、TCP 首部等功能。

Packet（分组）

一袋比特，由首部和主体组成。首部包含网络与网络栈决策所需信息；主体携带载荷，例如文件、图像或应用层首部。

Packet Switching（分组交换）

把数据分成分组，路由器/交换机检查各分组首部并独立处理每个分组的传输方式。

Path Vector Routing（路径向量路由）

类似距离向量路由，但向邻居通告的不是最短距离，而是到各目的地的完整路径。

Payload（载荷）

分组携带的数据。

Peer Table（邻居表）

路由器保存各个对等方或邻居所发信息副本的数据结构。

Poison Reverse（毒性逆转）

缓解计数到无穷的方法：向去往某目的地路径上使用的邻居通告自己到该目的地距离无穷。例如 A 给 C 的临时向量副本中，把所有经 AC 链路到达的目的地距离都设为无穷。

Port, Logical（逻辑端口）

操作系统分配给套接字、用于标识该套接字的编号。

Port, Router（路由器端口）

经链路把一台路由器连接到另一台路由器的物理端口。

Prefix Aggregation（前缀聚合）

用公共前缀把多条路由表项合成一条，例如把 101 和 100 合成 10*。

Prefix Tree（前缀树）

表示 IP 地址查找中比特匹配过程的二叉树，也就是查找表的遍历结构。

Reliability（可靠性，见健壮性）

两种解释：一是网络快速从故障中恢复，让未被分区隔开的端点能够通信；二是网络故障不干扰端点语义。

Reliable Delivery（可靠交付）

在尽力交付之上构建可靠传输服务。

Reliable Transport（可靠传输）

当且仅当传输机制会重发所有丢失或损坏的分组，并会尝试持续取得进展时，才称其可靠。

Resource Accountability（资源问责）

知道谁在使用哪些资源（带宽）、从而追究其使用责任的能力；这是互联网架构的一项缺失。

Robustness（健壮性，见可靠性）

只要网络没有被分区，两台主机最终就应能通信，而且故障绝不应干扰应用语义。

Route Aggregation（路由聚合）

不为每台主机保存一条转发表项，而为具有相同前缀且都走同一端口的一组主机保存一条。

Route Poisoning（路由毒化）

缓解网络不一致的过程：A、B 间链路失效时，B 向所有邻居通告自己不再有到 A 的链路，即距离无穷，表示无法到达 A。

Routing（路由）

引导分组从源到目的地，可用链路状态、距离向量、生成树等多种方式。它本质上是必须具备扩展性的全局过程，在控制平面执行，可以较慢。

Routing Table（路由表）

类似转发表，但可指路由器掌握的全部信息（包括其他邻居提供的信息），而非只有最佳转发表项。

Slash notation（斜杠表示法）

表示子网的记法，例如 1.2.0.0/10，其中 1.2.0.0 的前 10 位是子网前缀。

Sliding window（滑动窗口）

出于效率考虑，停止继续发送前允许在途且未确认的有限分组数量。

Socket（套接字）

操作系统用于把进程连接到网络栈的机制。

Soft State（软状态）

允许已存知识在可能变化或失效的假设下超时。软状态系统会周期性“忘记”信息，再通过重新请求或等待新消息学习。DHCP 租期、ARP 缓存超时、距离向量路由的周期消息都是例子。

Spanning Tree Protocol（生成树协议）

分布式协议：节点 X 的交换机发送格式为 (Y,d,X) 的消息，提议 Y 为根并通告到 Y 的距离 d。协议识别 ID 最小的节点，并以它为根构建生成树。

Split Horizon（水平分割）

功能与毒性逆转相同，但表达方式不同。用于完整更新时，路由器不会向自己用于到达目的地 X 的那个邻居通告任何到 X 的路由。

Statistical Multiplexing（统计复用）

各条流最大速率之和，大于把这些流合并后观察到的最大速率。

Subnet（子网）

本课指由第 2 层连接并共享同一网络地址的那部分网络。

Time to Live, TTL（生存时间）

在 IP 中指分组被丢弃前可经过的跳数，用于防止环路；更一般地说，TTL 是某项内容（如缓存表项）失效前的剩余时间。

Valid Routing State（有效路由状态）

在不复制分组时，当且仅当没有环路、没有死路，路由状态才有效；若会复制分组，则要求至少一个副本不会走入死路。

WAN（广域网）

可指任意第 3 层网络（不只是局域网），也可指跨越较大地理距离的网络（不是数据中心内部网络）。
