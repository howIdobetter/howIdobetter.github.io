ARP：连接第 2 层和第 3 层

连接第 2 层和第 3 层

回想一下，当数据包沿着堆栈向下移动到较低层时，它们会被附加的标头包裹起来。要发送 IP 数据包，我们首先在第 3 层填写其目标 IP。然后，我们将该数据包向下传递到第 2 层，在第 2 层中我们必须添加 MAC 地址才能沿链路发送数据包。我们添加什么 MAC 地址？

首先，我们需要检查目标 IP 是我们自己的本地网络中的某个人，还是另一个本地网络中的某个人。为了确定这一点，发件人的转发表将有一个条目指示本地 IP 地址的范围，有时称为我们的子网。例如，该条目可能会说 192.0.2.0/24 是直接的，这意味着 192.0.2.0 和 192.0.2.255 之间的所有地址都位于同一本地网络上。该表还有一个默认路由，表示所有其他非本地目的地都应转发到路由器。

如果目标 IP 在我们的子网中，我们需要某种方法在目标 IP 地址和该机器相应的 MAC 地址之间进行转换。如果目的地在我们的子网之外，我们需要某种方法将路由器的 IP 地址（来自转发表）转换为其相应的 MAC 地址，以便我们可以将数据包发送到路由器。

一种简单的解决方案是广播每个数据包，以便目的地或路由器一定会接收并处理它。然而，这是低效的。它迫使每个人解析每个数据包（例如读取第 3 层标头）以检查该数据包是否适合他们。此外，如果第 2 层网络具有多个链路，则第 2 层交换机必须在所有链路上泛洪数据​​包。

更好的方法是将目标 IP 地址转换为其相应的 MAC 地址（如果是本地）或路由器的 MAC 地址（如果非本地），并在第 2 层单播数据包。

ARP：地址解析协议

**ARP（地址解析协议）**允许机器将 IP 地址转换为其相应的 MAC 地址。

为了请求翻译，机器可以广播请求消息：“我有 MAC 地址 f8:ff:c2:2b:36:16。IP 192.0.2.1 的机器的 MAC 地址是多少？”

所有不是该 IP 地址的计算机都会忽略该消息。拥有此 IP 地址的用户单播回复发送者的 MAC 地址，内容为“I am 192.0.2.1, and my MAC address is a2:ff:28:02:f2:10”。

机器还可以向每个人广播自己的 IP 到 MAC 映射，即使没有人询问。

当您收到 IP 到 MAC 的映射时，您可以将其添加到本地 ARP 表，该表会缓存这些映射以供将来使用。该表还包括每个条目的到期日期，因为 IP 地址不会永久分配给计算机。不同的计算机可能会被分配相同的 IP 地址，或者同一台计算机可能会更改 IP 地址。 （TODO：接口？）

连接 ARP 和转发表

回想一下，在路由器的转发表中，我们有时会包含一个条目，指示主机直接连接到路由器。

实际上，路由器的转发表包含单个条目，将整个子网的 IP 地址范围直接映射。如果路由器收到目的地在此本地范围内的数据包，则路由器运行 ARP 查找相应的 MAC 地址，并使用二层将数据包发送到链路上的正确主机。

当多个主机连接在同一链路上时，这也对我们有帮助。在我们的概念图中，我们会说主机 A 直接连接在端口 1 上。该链路上可能有多个主机，因此通过使用 ARP，我们可以创建一个第 2 层数据包，该数据包仅单播到主机 A，而不是链路上的其他计算机。

给定一个将 192.0.2.0/24 等子网映射为直接的转发条目，我们如何确定给定的 IP 地址是否属于该范围？这是使用网络掩码而不是斜杠表示法来写入范围的有用之处。回想一下，要将此范围写入网络掩码，我们将所有固定位设置为 1，将所有非固定位设置为 0，以获得 255.255.255.0。然后该范围表示为 192.0.2.0，网络掩码为 255.255.255.0。

现在，为了检查地址是否在范围内，我们对地址和网络掩码执行按位与操作。这会导致所有未固定的低位被清零，仅保留固定的高位。然后，我们检查结果是否匹配 192.0.2.0（范围中的第一个地址，其中所有未固定位均为 0）。

请注意，当数据包跨跃点转发时，第 2 层目标将更改为下一跃点的 MAC 地址，以便数据包可以跨链路传输。但是，每跳的第 3 层目标保持不变。

IPv6 中的邻居发现

ARP 将 IPv4 地址转换为 MAC 地址。为了将 IPv6 地址转换为 MAC 地址，我们使用称为“邻居发现”的类似协议。

邻居发现不是广播 IP 到 MAC 转换的请求，而是将请求多播到特定组，并且每台计算机根据其 IP 地址侦听特定组。例如，IP 地址以 12:3456 结尾的每个人都可以侦听组 MAC 地址 33:33:FF:12:34:56，而 IP 地址以 78:90AB 结尾的每个人都可以侦听组 MAC 地址 33:33:FF:78:90:AB。

如果我想要与 IPv6 地址以 12:3456 结尾的用户对应的 MAC 地址，我可以将这些 IPv6 位插入组 MAC 地址以获得 33:33:FF:12:34:56，并且我知道具有该 IP 地址的用户一定正在侦听该组 MAC 地址。

一些术语：在邻居发现协议中，映射请求称为邻居请求，包含映射的回复称为邻居通告。
