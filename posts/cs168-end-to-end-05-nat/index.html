NAT：网络地址转换

动机：IPv4 地址耗尽

回想一下，我们只有 $$2^{32}$$ 个不同的 IPv4 地址，这不足以寻址 Internet 上的每个主机。我们已经看到 IPv6 是解决 IPv4 地址耗尽问题的强大解决方案，但 IPv6 的采用速度相当缓慢。

同时，为了节省地址，请回想一下，IANA 分配了特殊的 RFC 1918 专用 IP 地址范围，任何不需要 Internet 地址的网络都可以使用这些地址：192.168.0.0/16、10.0.0.0/8 和 172.16.0.0/12。事实证明，这些地址也经常在您的家庭网络中使用，因此您自己的个人设备不需要唯一的 IP 地址。但是，您确实需要访问互联网，那么如何使用私有IP地址呢？

NAT：概念

在 NAT 中，目标是使用单个公共 IP 地址来代表本地网络中的许多主机。诀窍是让网关路由器在发送消息之前将私有 IP 地址转换为单个公共地址。然后，路由器将公共地址转换回专用地址以用于传入回复。

爱丽丝、鲍勃和查克都在乔的轮胎店工作。它们有私有 IP 地址 A、B 和 C，这些地址不能在广泛的 Internet 上使用，因为它们不是唯一的。相反，Joe’s Tire Shop 中的每个人都必须共享一个公共 IP 地址，这是他们拥有的唯一唯一的、公开可理解的 IP 地址。

Alice 想要向具有公共 IP 地址 S 的外部公共服务器发送消息。她发送了一个数据包，内容为“发件人：A，收件人：S”。如果我们天真地发送这个数据包，S 将无法发送回复，因为 A 是一个私有 IP 地址。

相反，当数据包到达网关路由器时，它会将标头重写为“From：R1，To：S”。路由器还做了一个注释：如果我收到 S 的任何回复，它们应该发送给 A。

现在，当 S 收到数据包时，它可以向公共地址 R1 发送回复：“From：S，To：R1”。当网关路由器 R1 收到回复时，它会检查其注释，并将标头重写为“From：S，To：A”。然后，数据包被发送回 A。

现在，Alice、Bob 和 Chuck 都可以发送传出数据包。当路由器接收到数据包时，它必须记住外部目的地和内部发送者之间的映射。 （“B 刚刚向 N 发送了一个数据包，因此来自 N 的任何回复都应该发送回 B。”）

如果 Alice 和 Bob 都想与 S 交谈，就会出现一个问题。

如果回复从 S 到达，我们现在会产生歧义。路由器应该将此回复发送给 A 还是 B？

我们可以通过使用第 4 层的逻辑端口来解决这个问题。Alice 的连接说：“从：A，端口 50000，到：S，端口 80。”路由器将其重写为“来自：R1，端口 50000，至：S，端口 80”，就像以前一样。该注释现在表示，如果我收到从 S（端口 80）到 R1（端口 50000）的任何回复，它应该发送至 A。

Bob 可以创建一个单独的连接：“从 B，端口 60000，到：S，端口 80。”路由器将其重写为“从：R1，端口 60000，到：S，端口 80”，就像以前一样。此连接的注释说，如果我从 S、端口 80 到 R1、端口 60000 收到任何回复，则应该发送至 B。

更一般地说，路由器现在使用源 IP、目标 IP、协议、源端口和目标端口的 5 元组来跟踪连接。当路由器收到传出数据包时，会将私有源 IP 更改为公共源 IP，并记录 5 元组。然后，当路由器收到传入数据包时，它会查找该数据包属于哪个连接，并将数据包发送到适当的客户端（及其私有 IP）。

重写客户端端口号

我们还有最后一个问题：如果 Alice 和 Bob 都选择相同的端口号（例如端口 50000）而不是端口 50000 和端口 60000，该怎么办？

现在，路由器记住两个连接：（A 端口 50000 到 S 端口 80）和（B 端口 50000 到 S 端口 80）。如果路由器收到传入数据包“来自：S，端口 80，至：R1 端口 50000”，则该数据包是来自 A 还是 B 的连接是不明确的。

我们必须做的最后一个修复是还允许路由器重写端口号。当 Bob 发送“From：B，端口 50000，To：S，端口 80”时，路由器意识到其他人已经有使用端口 50000 到 S 端口 80 的连接。因此，路由器为 Bob 制作了一个“假”端口号（我们使用 60000），并重写源 IP 和源端口以获得：“From：R1，端口 60000，To：S，端口80。”

和以前一样，路由器会记住活动连接（A 端口 50000 到 S 端口 80），但对于 Bob，路由器还记录了假端口号：（B 端口 50000，伪造为 60000，到 S 端口 80）。

现在，如果路由器收到传入数据包“来自：S，端口 80，至：R1，端口 50000”，则这一定是发给 Alice 的。相比之下，像“From：S，端口 80，To：R1，端口 60000”这样带有假端口号的传入数据包，这一定是给 Bob 的。

请注意，Bob 不知道路由器正在更改他的端口号。当路由器将此数据包转发回Bob时，必须将假端口号更改回原始端口号。 “From：S，端口 80，To：R1，端口 60000”必须重写为“From：S，端口 80，To：R1，端口 60000”。更一般地说，任何私人客户端都不需要知道或关心他们的数据包被重写。路由器应该给他们所有人一种错觉，他们正在从他们的私有 IP 地址和他们选择的任何端口发送和接收数据包。

NAT：实施

当家庭路由器第一次连接到 ISP 时，它可以运行 DHCP 来接收 IP 地址。 （之前，我们讨论了运行 DHCP 的主机，但路由器也可以运行 DHCP。）ISP 的 DHCP 服务器进行回复并向家庭路由器分配单个 IP 地址。这是该路由器家庭网络中的所有主机将共享的单个公共地址。

NAT 有多种不同的模式。我们刚刚看到的称为端口地址转换（PAT），它使我们能够引入我们看到的假端口号。 PAT 模式要求路由器了解第 4 层协议，以便它们可以解析数据包、跟踪连接并重写标头。

PAT 是最复杂且使用最广泛的 NAT 模式，但也存在用于一对一地址转换的更简单的 NAT 模式。如果每个主机实际上都有自己的 IP 地址，但它们从私有地址发送数据包，则路由器可以进行一对一转换，将 10.0.0.1（私有）映射到 42.0.2.1（公共），将 10.0.0.2（私有）映射到 42.0.2.2（公共），依此类推。这种更简单的模式不会让我们通过将多个主机隐藏在单个公共地址后面来节省 IP 地址，但它在其他情况下仍然很有用。

NAT 在哪里使用？

NAT 增加了路由器转发数据包的复杂性。除了第 3 层标头之外，路由器现在还必须能够解析第 4 层标头。此外，路由器必须能够重写第 3 层和第 4 层标头。最后，路由器必须维护一个连接状态表来跟踪所有经过路由器的流量。所有这些功能增加了转发每个数据包所需的 CPU 周期数，并且还增加了每个流在路由器上所需的内存量。

由于 NAT 增加了路由器的复杂性，因此它尽可能靠近网络边缘执行，以限制通过路由器的流量数量。在家庭路由器上运行 NAT 是个好主意，因为家里没有太多设备会通过家庭路由器发送连接。相比之下，在高性能数据中心路由器上运行 NAT 并不是一个好主意。

实际上，即使在今天，几乎每个个人（家庭/办公室）网络都使用小规模 NAT 来支持 IPv4。由于 IPv4 地址耗尽，ISP 无法向每个客户（即每个家庭路由器）提供一个公共地址。因此，ISP 网络本身还必须运行更复杂的 NAT 版本，称为运营商级 NAT (CGNAT)。此版本的 NAT 部署在网络的更深处，并且需要路由器跟踪更多的连接。

请注意，我们通常不会对 IPv6 使用 NAT，因为有足够的 IPv6 地址可以为世界上的每台计算机分配唯一的公共地址。

入站连接

到目前为止，我们假设连接始终由具有私有 IP 地址的客户端发起。换句话说，第一个数据包总是从客户端传出到服务器。这与大多数家庭网络的运行方式一致。当您在浏览器中加载网站时，您就是发起连接的客户端。通常情况下，其他人不会尝试与您建立联系。

但是，如果您正在运行一台服务器，并且您确实希望外部世界的人们能够启动与该服务器的连接，该怎么办？外部用户无法将数据包发送到私有 IP 地址。他们可以尝试将数据包发送到路由器的 IP 地址，但如果路由器收到类似“发件人：外部用户，收件人：R1，端口 28”的数据包，则路由器不知道要将此数据包转发到哪个专用客户端。这是新连接的第一个数据包，因此路由器的表还没有有关此连接的信息。

为了允许入站连接，执行 NAT 的路由器需要端口映射表。 Alice 位于网络内部，只有一个私有 IP 地址，它告诉路由器：我将运行一个新服务器，并侦听端口 28 上的请求。现在，如果路由器看到某个外部用户发送到 R1 端口 28 的数据包，路由器就会知道将此数据包转发给 Alice。

此端口映射表中的条目可能需要手动指定（例如，Alice 手动配置路由器）。 UPnP（通用即插即用）和 NAT-PMP（NAT 端口映射协议）等动态协议允许动态配置开放端口。这些协议有时由需要入站连接的在线游戏等应用程序使用。

NAT 的安全影响

NAT 破坏了端到端原则。到目前为止，我们已经说过，通过第 3 层，互联网上的任何人都可以联系到其他人。但是，由于 NAT 默认情况下不允许入站连接，因此无法自动联系家庭网络中仅拥有私有 IP 地址且共享公共 IP 地址的用户。他们需要先配置路由器，然后才能接受入站数据包。

NAT 具有默认情况下不允许入站连接的属性。这可以被视为一项安全功能，尽管它更多的是副作用而不是有意的设计功能。默认情况下，NAT 会导致入站连接被阻止，这可能有助于阻止攻击者尝试连接到网络内的主机。这种行为实际上与防火墙非常相似（有关更多信息，请参阅加州大学伯克利分校 CS 161 注释），防火墙默认情况下也经常阻止入站连接。也就是说，这主要是巧合，因此 NAT 并没有真正实施有原则的安全策略，并且不应被视为万无一失的防御。

NAT 还有一个副作用，即它可以帮助保护客户端隐私。再次强调，这并不是一个真正有意的安全功能。由于路由器重写了客户端的 IP 地址，因此当服务器收到数据包时，它不知道原始发送者的身份（可能是 Alice、Bob 或 Chuck）。

相比之下，如果我们不使用 NAT，服务器就可以获知发送者的确切身份。另外，如果我们不使用 NAT 而使用 IPv6，服务器也许能够了解发送者正在使用的确切计算机，因为 IPv6 地址有时是使用 MAC 地址自动配置的（将 MAC 地址位复制到 IP 地址中）。如果我们使用 IPv6 并且仍然需要客户端隐私，那么确实存在一些替代解决方案，例如 IPv6 临时/隐私地址。
