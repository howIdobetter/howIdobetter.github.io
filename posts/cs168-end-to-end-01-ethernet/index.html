以太网

本地网络

在本节中，我们将重点关注局域网内部发生的情况，例如您家里的计算机和家庭路由器的网络。这与我们迄今为止看到的跨越更长距离的广域网形成鲜明对比。

特别是，我们将研究第 2 层的转发和寻址。我们必须定义如何将数据包从本地主机转发到路由器。我们还将了解同一本地网络中的主机如何在第 2 层交换消息，而根本不需要联系路由器。第 2 层的主要协议是以太网。

连接本地主机

到目前为止，我们已经绘制了恰好连接两台机器的链接。在本地网络中，我们画了一条线将每个主机连接到路由器。

实际上，一根电线可用于连接多台机器。在本地网络中，主机和路由器可以都在同一线路上。我们可以进一步抽象，并注意到在第 2 层，路由器实际上只是一台像任何其他机器一样的机器（恰好在更高层运行路由协议）。最终，线路并不真正关心连接的机器如何处理它们交换的数据。

在本地网络中连接计算机的最佳方法是什么？早些时候，当我们第一次引入路由时，我们考虑使用网状拓扑来连接世界上所有的计算机对。我们还考虑使用一根电线连接所有计算机。最终，我们认为对于全球网络来说，这两种方法都不实用，我们需要引入路由器。

我们可以在本地网络中再次考虑这些拓扑。网状拓扑仍然相当不切实际。如果有新主机加入，我们必须添加一条电线将其连接到所有其他主机。然而，总线拓扑（我们通过一根电线连接所有计算机）在本地网络中非常常见且实用。

单线总线拓扑引入了共享媒体的概念。当我们绘制连接两台机器的链接时，只有那两台计算机使用该链接进行通信。现在，从 A 到 C 的数据包和从 B 到 D 的数据包可能同时在线路上，并且该线上的电信号无法同时容纳两个数据包。

打个比方，考虑多人进行群组通话，共享一条电话线：任何两个人都可以互相交谈，但不能同时进行两个对话，否则没人能理解所说的内容。

为了简单起见，我们将链接绘制为带有电信号的电线，但实际上，链接技术可以使用其他共享媒体。例如，在无线链路技术中，通过链路连接的所有主机共享电磁频谱的相同部分。

通过共享媒体进行沟通：协调方法

在具有共享介质的网络中，存在来自不同节点的传输可能相互干扰或冲突的风险。如果两台计算机尝试同时传输数据，它们的信号将会重叠和干扰。接收者可能无法解码信号，并且无法分辨是谁发送了信号。为了解决这个问题，我们需要一个多访问协议来确保多台计算机可以共享链接并通过它进行传输。

一类可能的方法是将固定部分的资源分配给链路上的每个节点。我们可以考虑通过两种方式分配资源。在频分复用中，我们为每台计算机分配不同的频率片。 （考虑 AM/FM 无线电或广播电视，它们将频率划分为频道。）在时分复用中，我们将时间划分为固定时隙，并为每个连接的节点分配一个时隙。

固定资源分配有一些缺点。分发的频率/时间有限。此外，并不是每个人一直都有话要说，因此我们分配的频率/时间可能大部分时间都没有被使用。这种方法很浪费，因为它将计算机限制在其特定分配的片上，即使其他片可能未被使用。

另一类方法不是固定分配，而是基于节点轮流，没有任何固定分配。在此类别中，我们按时间进行动态分区，以便节点仅使用轮到其所需的时间，而不会浪费时间。我们可以考虑通过两种方式让节点轮流运行。

在轮询协议中，集中协调器决定每个连接的节点何时开始发言。协调员一一去各个节点询问该节点是否有话要说。如果节点同意，协调器会让节点发言一段时间。如果节点拒绝，协调器立即移动到下一个节点，并且该节点不会浪费任何资源。蓝牙是使用这个想法的现实世界协议。

让节点轮流的另一种方法是令牌传递。我们没有中心化的协调器，而是有一个可以在节点之间传递的虚拟令牌，并且只有拥有令牌的节点才被允许说话。如果一个节点有话要说，它会在传输时保留令牌，然后将其传递给下一个节点。如果一个节点此时没有什么可说的，它会立即将令牌传递给下一个节点。 IBM 令牌环和 FDDI 是使用此想法的协议的现实示例。

这些回合制方法的一个缺点是复杂性。我们必须实现某种形式的节点间通信，这可能会变得复杂。在令牌传递中，我们可能需要一些专用的频率通道供节点在彼此之间可靠地传递令牌。我们可能还必须处理复杂的情况，例如两个节点都认为自己拥有令牌并导致冲突。在轮询协议中，我们需要指定一个中央协调器来与节点通信，并实现协调器与节点通信的方式。在蓝牙中，您的智能手机可以是与辅助设备通信的中央协调器，但在其他网络中，协调器是谁可能并不明显。

通过共享媒体进行通信：随机访问方法

除了固定分配或轮流之外，第三类方法是随机访问。在这种方法中，我们只允许节点在有话要说时进行交谈，并在冲突发生时进行处理。节点之间不协调，只要有东西要发送就发送数据。

随机访问协议的一大优点是简单。与回合制方法不同，我们不需要实现节点间通信。

当接收者收到数据包时，它会回复一个确认。如果两个节点同时发送数据，则冲突会导致其数据包损坏，因此不会发送 ack。如果发送者没有看到确认，它会等待一段随机的时间并重新发送。等待一段随机的时间，而不是立即重新发送，可以帮助我们避免重新发送数据包时发生冲突。

朴素的随机访问协议是“粗鲁的”，因为节点随时开始通话，然后处理冲突。该协议的一个更“礼貌”的变体称为载波侦听多路访问（CSMA）。节点首先监听共享媒体，看看是否有人在说话，只有在安静时才开始说话。这里，“听”是指感测线路上的信号。

请注意，CSMA 并不能帮助我们避免所有冲突。如果信号沿线路的整个长度瞬时传播，则 CSMA 中不会发生冲突。然而，传播延迟可能会带来问题。假设线路一端的节点 A 听到静音并开始传输。信号可能尚未传播到线路另一端的节点 B。节点 B 听到静默后也开始传输，从而引起冲突。

此二维图演示了传播延迟如何导致冲突。水平横截面显示了电线在某一时刻的情况，让我们看到信号在该时刻在电线上传播了多远。垂直横截面显示了电线上随时间变化的单个位置，让我们可以看到该位置何时看到传输的第一个和最后一个位。 H2 和 H4 在开始传输之前都会听到寂静，但它们的信号仍然发生冲突。

为了缓解这个问题，我们可以使用CSMA/CD（带冲突检测的载波侦听多路访问），它扩展了 CSMA 的思想。除了先听再说外，我们还边听边说。如果您在传输时开始听到声音，您会立即停止。请注意，CSMA/CD 仍然不能解决冲突问题，但它允许我们更快地检测到冲突。

如果只有一个扬声器，就不会发生任何冲突，并且我们所有的随机访问方案都应该可以正常工作。如果只有几个扬声器，可能会偶尔发生冲突，但我们所有的方案都可以处理它们。但是，如果许多发送者想要同时通话，我们可能会遇到重复冲突的问题，并且等待随机时间重新发送也无济于事。

为了处理重复的冲突，CSMA/CD 使用二进制指数退避。每次我们在重传尝试中检测到冲突时，我们都会在下一次重传之前等待两倍的时间。请注意，我们仍然随机选择重传时间，但每次检测到冲突时，我们都会从限制范围为两倍高的范围中选择随机数。例如，如果我们选择 [0, 4] 范围内的随机时间并检测到碰撞，则我们选择的下一个随机时间将在 [0, 8] 范围内。

二进制指数退避在这两种情况下都效果很好。当有几个节点说话时，重复的冲突并不常见，因此我们可以在短暂的等待时间后重新传输。当有很多节点说话时，就会有很多重复的冲突，因此延迟会呈指数增长，直到没有冲突为止（例如，足够多的节点已经延迟到很远的将来，而现在竞争的节点较少）。这种方法确保我们只在许多节点想要说话时放慢速度，并在少数节点想要说话时保持快速传输。

第 2 层简史：ALOHANet

1968 年，诺曼·艾布拉姆森 (Norman Abramson) 在夏威夷大学遇到了麻烦。夏威夷大学有一台中央计算机，他需要一种方法让其他岛屿上的计算机访问这台中央计算机。由此产生的设计对现代第 2 层协议设计非常有影响。

由此产生的协议被称为 ALOHANet（夏威夷地区在线附加链接），它允许从其他岛屿到中央计算机的无线通信。 ALOHANet 是无线的并使用共享介质，每个人都通过同一链路发送数据。

由于其非对称设置，ALOHANet 使用固定分配和随机访问的组合。中央计算机（集线器）使用自己的专用频率来传输传出消息，所有远程节点都监听该频率以接收消息。由于专用频率上只有一个发送器，因此不存在冲突风险。

相比之下，所有远程节点都在单独的共享频率上进行传输，并且集线器监听该频率。集线器不会与远程节点发生冲突，因为它们使用不同的频率，但远程节点可能会相互冲突。

这种不对称设计对于 ALOHANet 来说效果很好，因为集线器可能比远程节点要发送更多内容。

ALOHANet 是最早使用随机访问协议来处理冲突的系统之一，这种方法后来被用于以太网。 ALOHANet 使用天真的粗鲁方法进行随机访问。后来的协议（例如以太网）使用了更礼貌的 CSMA/CD 方法，我们在传输之前和传输过程中侦听冲突，并在发生冲突时以指数方式后退。

LAN 通信：MAC 地址

由于多台计算机可以沿着同一条以太网链路连接，因此我们实际上可以使用第 2 层协议在同一链路上的本地计算机之间发送消息，而根本不需要使用任何第 3 层协议（例如，没有路由器转发数据包）。以邮政系统类比，同一个房间的两个人可以互相传递信件，而无需将信件发送到邮局。

通过共享媒体发送消息的一个问题是：当我们传输消息时，链接上的每个人都会收到消息，而不仅仅是预期的收件人。为了仅向一个人发送消息，我们需要第 2 层的寻址系统，以便我们可以识别消息要发送给哪台机器。以邮政系统为例，如果我在一个房间里讲话，每个人都会收到消息。要与某个特定的人交谈，我需要使用他们的名字来提及他们。

在第 2 层，每台计算机都有一个 MAC 地址（媒体访问控制）。 MAC 地址长 48 位，通常以十六进制书写，每 2 个十六进制数字（8 位）用冒号分隔，例如f8:ff:c2:2b:36:16。 MAC 地址有时称为以太地址或链路地址。

MAC 地址通常永久硬编码（“烧录”）在设备（例如计算机中的 NIC）上。大多数操作系统都会允许您覆盖软件中的 MAC 地址，但每个设备都已安装了 MAC 地址。 MAC 地址是根据创建硬件的制造商分配的。前两位是标志，接下来的 22 位标识制造商，最后 24 位标识该制造商地址空间内的特定机器。

为什么不直接使用 IP 寻址呢？链接上的主机可能想要交换消息，而无需连接到 Internet（即它们根本没有 IP 地址）。

这种永久寻址方案与 IP 不同，在 IP 中，您首次加入网络时会收到一个地址，该地址取决于您的地理位置。 MAC 地址通常应该是全局唯一的，因为您可以将计算机插入任何本地网络，如果链接上的两台计算机具有相同的 MAC 地址，那就不好了。

LAN 通信类型、以太网数据包结构

第 2 层数据包中有不同的可能目的地。在单播中，数据包旨在发送给单个接收者。在广播中，数据包适用于本地网络上的所有计算机。在多播中，数据包适用于本地网络中属于特定组的所有计算机。机器可以选择加入某些组来接收发送给该组的数据包。以太网支持单播、组播和广播。

请注意，广播有时被认为是多播的特殊情况，其中每个人都自动成为广播组的一部分。

这种单播/广播/组播模型也扩展到其他层。例如，我们在第 3 层看到了任播，其目标是发送到组中的任何一个成员（具有相同 IP 地址的任何服务器）。

以太网数据包结构

以太网中的数据包称为帧。许多字段看起来与 IP 标头字段相似，但也存在一些差异。

以太网数据包以 7 字节前导码开头，表示数据包的开始。这有助于在通过线路发送信号时将数据包分开。

然后，我们就有了目标和源 MAC 地址，类似于 IP 标头中的目标和源字段。我们有一个 2 字节类型字段，它允许我们在 IPv4 或 IPv6 之间进行解复用，并将数据包有效负载传递到正确的下一个协议。这类似于 IP 标头中的协议字段或 TCP/UDP 标头中的端口字段。我们还有一个校验和，但与 IP 不同，校验和是针对整个数据包的，因此我们不必依赖更高层（例如，数据包可能根本不是 TCP/IP）。

为了单播消息，我们将目标 MAC 地址设置为特定机器的 MAC 地址。共享介质上的每个人都会收到数据包，因此每个人都需要检查目标 MAC 以查看该数据包是否是发给他们的。如果目标 MAC 地址与您的地址不匹配，您应该忽略该数据包。

为了广播消息，我们将目标 MAC 设置为特殊地址 FF:FF:FF:FF:FF:FF（全 1）。就像在单播中一样，共享介质上的每个人都接收数据包，但这一次，因为目标 MAC 地址是广播地址，所以每个人都知道读取数据包。请注意，这个全 1 广播地址在每个以太网中都是相同的。

为了多播消息，我们将目标 MAC 设置为该组的地址。回想一下，MAC 地址的前两位是标志。分配给机器的普通地址始终将第一位设置为 0，而组地址始终将第一位设置为 1。就像单播和广播一样，每个人仍然会收到消息。属于某个组的任何人都需要确保他们正在侦听该组的地址，以便接收多播到该组的数据包。需要附加协议来控制谁属于哪些组，我们不会进一步讨论它们。

带以太网的第 2 层网络

到目前为止，我们已经展示了第 2 层协议在连接有多台计算机的单个链路上运行，但我们可以引入多个链路并完全使用第 2 层构建网络。数据包可以转发，机器甚至可以运行路由协议，所有这些都专门使用第 2 层 MAC 地址。

我们在 IP 层运行的路由协议也可以在第 2 层运行，但缺点是我们无法聚合 MAC 地址。 IP 地址是根据地理位置分配的，但 MAC 地址是根据制造商分配的，因此没有明确的方法来聚合它们。这个缺点就是我们无法仅用第 2 层构建全球互联网的原因。

如果单个本地网络中有多个链路，我们必须确保如果有人广播消息，第 2 层的任何交换机都会将数据包从所有传出端口转发出去。

在具有多个链路的第 2 层网络中，组播变得更加复杂。需要额外的协议，但我们不会进一步讨论。

Bonjour/mDNS 是 LAN 上有用的多播示例，它是 Apple 开发的一种协议。在此协议中，所有 Apple 设备（例如 iPhone、iPad、Apple TV）都经过硬编码以加入本地网络上的特殊组。如果您的 iPhone 想要找到附近的设备来播放音乐（例如 Apple TV、Apple 扬声器或 HomePod 或其他名称），iPhone 可以向群组多播一条消息，询问是否有人可以播放音乐。该组中的设备还可以多播响应，说“我是 Apple TV，我可以播放音乐”。有趣的是，该协议实际上还使用多播组中的 DNS 来发送 SRV 记录，该记录将每台机器映射到其功能。

历史记录：在现代互联网中，我们说过术语“路由器”和“交换机”是可以互换的。现在我们有了二层网络的概念，我们可以说交换机只工作在第一层和第二层，而路由器工作在第一层、第二层和第三层。

如果您回到包装和解开标头的图片，我们假设每个路由器都会解析数据包至第 3 层，并通过 IP 将数据包转发到下一个路由器。但是，如果我们有一个具有多个链路的第 2 层网络，交换机只需将数据包传递到第 2 层，然后通过以太网将数据包转发到下一个交换机。

如今，几乎所有交换机都实现了第 3 层，这就是我们交替使用这些术语的原因。从历史上看，以太网早于互联网，这就是交换机和路由器之间存在区别的原因。
